# CUI and CMMC Scope

**Important:** This book records architecture and assessment evidence. It does not declare or certify CMMC compliance.

The architecture must identify components that process, store, or transmit Federal Contract Information or Controlled Unclassified Information, plus components that provide security protection for those systems.

## Required architecture evidence

- System and authorization boundaries
- CUI categories and authoritative markings
- Asset and software inventory
- Data-flow and external-connection diagrams
- Human and service identities
- Access-control enforcement points
- Encryption and key-management paths
- Audit generation, collection, retention, and review
- Configuration and change-management path
- Backup, recovery, media, and disposal path
- Control implementation and evidence ownership

## Scope states

| State | Meaning |
|---|---|
| In scope | Component processes, stores, transmits, or protects CUI/FCI |
| Security protection asset | Protects an in-scope component or data flow |
| Specialized asset | Requires explicit treatment under the applicable assessment approach |
| Out of scope | Has no path to CUI/FCI and is separated by validated controls |
| Undetermined | Evidence is insufficient; treat as a discovery finding |

The contracts, security, and assessment teams must confirm the applicable contractual baseline and CMMC level.
